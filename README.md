# Darin

> 내니 매칭에서 시작해, 아이가 자라는 동안의 돌봄 기록을 쌓고, 그 데이터를 기반으로 정밀한 AI 양육 상담을 제공하는 통합 돌봄 플랫폼

Darin은 부모와 돌봄 제공자를 연결하는 매칭으로 시작합니다. 계약과 동시에 돌봄 기록이 시작되고, 신생아부터 영유아, 어린이집 시기까지 아이에 대한 데이터가 누적됩니다. 충분한 데이터가 쌓이면 그 가정에만 맞는 AI 상담이 가능해집니다. 일반적인 육아 챗봇과 달리, Darin은 그 아이의 실제 기록과 산모의 컨디션을 알고 답합니다.

## 해결하는 문제

- 부모는 신뢰할 수 있는 돌봄 제공자를 찾고 검증하기 어렵다.
- 돌봄 제공자는 일감을 카페 글이나 지인 소개에 의존해 일이 불안정하다.
- 매일의 돌봄 기록은 작성에 시간이 많이 들고, 흩어져서 한 아이의 성장 그림으로 이어지지 않는다.
- 육아 정보는 넘치지만, 정작 "우리 아이"에게 맞는 구체적인 답은 어디서도 얻기 어렵다.

## 핵심 가치

진입은 쉽게, 데이터는 깊게. 매칭은 사용자를 끌어들이는 입구이고, 일상 기록은 데이터를 쌓는 엔진이며, 누적된 데이터가 다른 서비스가 흉내 낼 수 없는 AI 상담의 근거가 된다.

## 타겟 사용자

| 페르소나 | 설명 | 핵심 니즈 |
|---|---|---|
| 산모 / 부모 | 첫 출산을 앞두거나 어린 자녀를 키우는 부모 | 믿을 만한 돌봄 제공자 매칭, 아이 하루의 정확한 파악, 우리 아이에 맞는 조언 |
| 케어기버 | 산후관리사, 내니, 베이비시터 | 안정적인 일감, 빠른 기록 작성, 신뢰 증명 |

## 양육 라이프사이클

Darin은 단일 시점의 도구가 아니라 양육 전 기간을 따라가는 플랫폼이다.

```
[ 매칭 ] -> [ 계약 ] -> [ 일상 돌봄 기록 ] -> [ 성장 / 데이케어 기록 ] -> [ 누적 데이터 ] -> [ AI 상담 ]
```

- **매칭 / 계약**: 조건 기반으로 부모와 케어기버를 연결하고 앱 내에서 표준 계약을 체결한다. 계약 확정이 곧 기록 기능의 시작 스위치다.
- **돌봄 기록**: 수유, 수면, 배변, 체온, 특이사항을 짧게 입력하면 AI가 부모용 리포트로 정리한다.
- **성장 / 데이케어**: 시기에 따라 이유식, 발달 이정표, 예방접종, 어린이집 적응, 잔병치레로 항목이 확장된다.
- **누적 데이터**: 성장 곡선, 수면 패턴, 식사량 추이, 질병 이력, 발달 속도, 산모 컨디션이 그 가정만의 데이터셋으로 쌓인다.
- **AI 상담**: 누적 데이터를 근거로 그 아이와 그 산모에게만 맞는 정밀한 답을 제공한다.

## 주요 기능

### 1. 매칭과 계약

- 부모 돌봄 요청서 작성 (기간, 입주 여부, 지역, 예산, 요구사항)
- 케어기버 프로필 (경력, 자격증, 검증 배지, 시급, 가능 시간)
- 조건 기반 매칭 및 앱 내 사전 인터뷰
- 표준 계약서 작성과 전자 서명

### 2. AI 돌봄 리포트

- 케어기버가 짧은 메모를 입력하면 AI가 자연스러운 부모용 리포트로 변환
- 전송 전 원본 메모와 AI 리포트를 나란히 확인해 부정확한 기록 방지

### 3. 아이별 활동 기록

- 수유, 수면, 배변, 활동, 기분, 건강, 특이사항을 시기별로 기록
- 성장 곡선과 패턴 추이를 시계열로 시각화

### 4. AI 양육 상담

- 누적된 아이 데이터와 산모 컨디션을 근거로 한 정밀 상담
- 일반론이 아니라 그 가정의 실제 기록을 인용한 구체적 질문과 답변

#### AI 상담 예시 (복직 시나리오)

부모의 질문에 AI는 일반적인 조언 대신 데이터를 근거로 확인 질문을 먼저 던지고, 답을 받으면 아이의 적응 기록, 모유수유 지속 여부, 산모의 수면과 피로 데이터, 어린이집 운영 정보를 엮어 맞춤형 답변을 제공한다. 예를 들어 어린이집 등원 직후 우는 시간이 줄어든 추이를 근거로 복직 타이밍을 평가하고, 근무 시간과 하원 시간의 공백을 짚어 추가 돌봄 연장을 제안하며, 산모의 최근 수면 부족 기록을 보고 본인 컨디션 관리까지 함께 권한다.

## MVP 범위

| 우선순위 | 기능 | 설명 |
|---|---|---|
| 1 | 매칭과 계약 | 요청서 작성, 프로필, 매칭, 계약 체결 |
| 2 | 아이 프로필 | 이름, 나이, 보호자 정보 설정 |
| 3 | 활동 메모 입력 | 수유, 수면, 배변, 기분, 특이사항 |
| 4 | AI 돌봄 리포트 생성 | 짧은 메모를 부모용 리포트로 변환 |
| 5 | 리포트 저장과 히스토리 | 아이별 기록 누적 |

가장 중요한 데모는 매칭에서 계약으로 이어지는 순간과, 짧은 메모가 부모용 리포트로 바뀌는 순간이다.
